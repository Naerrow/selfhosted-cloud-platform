# decision-log-archive

본 디렉터리는 [`../decision-log.md`](../decision-log.md)의 **아카이브 보관소**다.

## 아카이브 정책

[`../decision-log.md`](../decision-log.md) 머리말 + 내부 에이전트 지침의 컨텍스트 보존 정책에 따라, 다음 조건 중 하나라도 충족되면 본 디렉터리로 아카이브한다.

- **분기 종료** (3·6·9·12월 말)
- **decision-log.md가 200줄 초과**
- **자동 로드 대상(최근 2주치) 바깥의 누적분이 의미 있는 부피가 됨**

## 파일명 규칙

`YYYY-QN.md` (예: `2026-Q2.md`, `2026-Q3.md`). 분기를 가로지르는 항목은 시작 분기 파일에 둔다.

## 아카이브 후 처리

- decision-log.md 본문에는 아카이브 파일 링크만 남긴다 (한 줄).
- 본 디렉터리 인덱스(README.md)에 새 아카이브 파일을 한 줄 추가한다.

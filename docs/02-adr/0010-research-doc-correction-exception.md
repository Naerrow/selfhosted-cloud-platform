# ADR-0010: 문서 정정과 설계 변경을 구분한다

- **상태(Status)**: Accepted
- **결정일(Date)**: 2026-05-28
- **결정자(Decider)**: 프로젝트 메인테이너
- **관련 ADR**: 0001 (ADR 운영 체계), 0009 (내부 전용 산출물)

## Context — 맥락

본 프로젝트의 운영 규칙은 두 방향에서 압박을 받는다.

1. **사실 오류 전파 방지**: 추적 문서의 잘못된 사실(예: "Proxmox 32노드 한계")이 다른 문서로 전파되었을 때, 보호 규칙을 절대 불변으로 두면 정정 경로가 없다.
2. **ADR 인플레이션 방지**: ADR-0001은 "비자명한 결정"만 ADR화하라고 명시한다. 그러나 정정·결정·해석을 구분하는 명시 기준이 없으면 사소한 오타 정정도 ADR로 누적될 위험이 있다.

실제 운영에서 이 양극단이 모두 발생했다. (a) 사실 오류가 4개 문서까지 전파된 후 발견. (b) 사실 정정을 ADR Errata로 누적하다 보니 ADR 본문이 비대해지고, ADR 정정·신규 ADR·운영 정책 변경의 경계가 흐려짐.

해결책은 **"정정 vs 결정 변경 vs 정책 신설"을 단일 분류 체계로 구분**하고, 각 분류에 맞는 절차를 강제하는 것이다.

## Decision — 결정

추적 문서의 변경은 다음 **4단계 분류 체계**로 구분하고, 각 분류의 절차에 따른다.

### 분류 1 — 오타·문법·서식·깨진 링크

- **예시**: 맞춤법, 표 정렬, trailing whitespace, 인용 URL 경로 오타.
- **절차**: **그냥 수정**. ADR·decision-log 기록 모두 불필요. 커밋 메시지로 충분.

### 분류 2 — 1차 출처 기반 사실 정정 (결정 영향 없음)

- **예시**: 공식 매뉴얼·법령 원문과 본문이 충돌하는 경우. 문서 내부 자체 모순(본문 ↔ 요약 표 등). 인용 URL·출처 메타데이터의 깨짐·갱신.
- **절차**: **본문 직접 수정**. 필요하면 다음 중 선택해 기록.
  - 사전 조사 자료처럼 1차 출처 인용이 본문 가치의 일부인 문서: **머리말 개정 표기**에 한 줄 추가.
  - 일반 추적 문서: **`decision-log.md`에 한 줄 기록** (선택).
- ADR 자체는 작성하지 않는다.

### 분류 3 — 1차 출처 기반 사실 정정 (결정 근거·결론에 영향 있음)

- **트리거**: 정정으로 인해 다음 중 하나가 발생.
  - (a) 결정의 채택/거부가 뒤바뀜
  - (b) Alternatives 표의 우열이 역전됨
  - (c) Decision 섹션의 결론 문장이 다르게 쓰여야 함
- **절차**: **신규 ADR 작성** + 기존 ADR의 상태를 **Superseded by ADR-NNNN** 으로 갱신. 본문은 보존(이력 가치).

### 분류 4 — 운영 정책의 신설·변경

- **예시**: 산출물 공개 정책, 정정 절차 자체의 변경, 결정 기록 체계 변경.
- **절차**: **신규 ADR 작성**. 기존 정책 ADR이 있으면 Superseded로 연결.

### "결정 영향 있음"의 판단 기준 (분류 2 ↔ 3 경계)

판단이 흔들릴 때는 위 (a)(b)(c) 세 질문을 순서대로 묻는다.

- 세 질문 모두 "아니오" → 분류 2 (약점 표현·근거 표의 일부 항목 수정 등, 나머지 근거가 결정을 충분히 뒷받침)
- 하나라도 "예" → 분류 3
- 판단 불가 → 사용자에게 보고해 결정을 받는다.

### 적용 범위

- **대상**: ADR Accepted, 사전 조사 자료, vision·scope·glossary·personas·README·ROADMAP, 컴플라이언스 매핑·위협 모델 등 **모든 추적 문서**.
- **예외**: 내부 전용 산출물(예: 내부 에이전트 지침, 내부 요구사항 추적표 등)은 [ADR-0009](./0009-internal-only-design-docs.md) 정책의 지배를 받는다. 단, 내부 문서 안에서도 정정 자체의 분류 판단은 본 ADR의 4단계를 그대로 따른다.

## Alternatives Considered — 대안 비교

| 대안 | 강점 | 약점 | 결정 |
|---|---|---|---|
| **보호 규칙 절대 불변** | 원본 안정성 100% | 사실 오류가 다른 문서로 전파됨 (실제 발생), 정정 경로 부재 | 거부 |
| **모든 정정 시 신규 ADR** | 흔적 100% 보존 | ADR-0001이 경고한 **ADR 인플레이션** 발생, "결정 vs 정정" 본질 차이가 흐려짐 | 거부 |
| **분류 체계 없이 사례별 판단** | 유연성 | 판단 회색 지대 큼, 같은 유형이 다르게 처리되는 일관성 결함 | 거부 |
| **Errata 섹션을 ADR 본문에 누적** | 정정 흔적이 ADR 본문 안에 남음 | ADR 본문이 비대해지고, 사소한 정정까지 ADR 영역으로 끌어들임 | 거부 (이전 운영 방식, 본 ADR에서 폐지) |
| **사전 조사 자료 한정 예외 절차** | 명시적 범위 | ADR 본문 정정·일반 추적 문서 정정에 매핑되지 않음 (정책 공백) | 거부 (본 ADR의 이전 버전, 본 결정으로 일반화) |
| **4단계 분류 체계** | 일관 기준, 인플레이션 예방, 정정/결정 변경 경계 명료 | 분류 2/3 경계의 회색 지대는 잔존(판단 기준 (a)(b)(c)로 완화) | **채택** |

## Consequences — 결과

### 긍정적
- 정정·결정 변경·정책 신설이 단일 기준으로 분류되어 일관 처리된다.
- ADR 인플레이션이 원천 차단된다 — 분류 1·2가 ADR 영역에서 빠짐.
- 사실 오류 전파가 정정 경로 부재로 누적되는 일이 사라진다.
- ADR-0001의 "Accepted 불변" 규정이 분류 체계 안에서 합리적으로 완화된다.

### 부정적
- 분류 2 ↔ 3 경계의 회색 지대는 잔존. 판단 기준 (a)(b)(c)로 완화하나 완전 제거는 불가능.
- 정정 시마다 분류를 먼저 판단해야 하는 작은 인지 비용 발생.

### 중립적
- 본 ADR의 이전 버전("사전 조사 자료에 한정된 정정 예외")은 본 결정으로 흡수·일반화된다.
- ADR-0002·ADR-0003에 누적되어 있던 Errata 섹션은 본 ADR 채택과 함께 분류 2로 재분류되어 본문 직접 정정으로 전환된다 (정정 흔적은 git 이력과 decision-log로 보존).

## Operational Implications — 운영 영향

- 추적 문서 변경 작업 시작 전 항상 **분류부터 결정**한다.
- 분류 3 판단이 흔들리면 사용자에게 보고해 결정을 받는다.
- `decision-log.md` 기록은 분류 2 이상에서 권장, 분류 3·4는 의무.
- ADR-0001과 본 ADR이 ADR 변경 정책의 단일 권위 출처. 분쟁 시 두 ADR로 회귀한다.

## Open Questions

- 분류 2 ↔ 3 경계 판단을 보조할 체크리스트가 필요한가 → 분류 3 판단이 3회 이상 흔들리는 시점에 재검토.
- 외부 기여자 합류 시 본 분류 체계의 학습 곡선 → README 또는 별도 CONTRIBUTING.md에 한 줄 요약 게재 검토 (Phase 1 진입 시).
